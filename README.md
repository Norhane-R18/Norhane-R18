## How I Approach Projects

| Step | What I focus on |
|---|---|
| Understand the problem | Identify users, roles, needs and constraints |
| Design the system | Define entities, APIs, architecture and access rules |
| Build the backend | Implement business logic, validation and secure endpoints |
| Secure the application | Apply authentication, authorization and permission checks |
| Test and observe | Use Postman, logs, health checks and manual validation |
| Document the work | Explain features, stack, setup, security and future improvements |
